The VisionClient

Single entry point for everything. Flipping one boolean toggles between fixture-backed demo and live OCI calls.

from oci_vision import VisionClient # Demo mode (zero credentials) client = VisionClient(demo=True) report = client.analyze("dog_closeup.jpg") # Live mode (uses ~/.oci/config) client = VisionClient() report = client.analyze("oci://my-bucket/photo.jpg")

Pydantic v2 Models

  • AnalysisReport: primary return type carrying all feature results
  • ClassificationResult: labels + confidence scores
  • DetectionResult: bounding polygons per object
  • TextDetectionResult: extracted text lines + regions
  • FaceDetectionResult: face bounds + landmark maps
  • DocumentResult: fields, tables, key-value pairs
  • All models match the real OCI Vision API response shapes
  • Validated, serializable, ready for JSON/HTML rendering

Demo Mode

Oracle Database 26ai

Optional. Off by default. Flip one env var to get persistent run storage and semantic search over past analyses.

  • Run storage: every analysis persisted automatically
  • Semantic search: vector-powered queries over past results
  • Docker Compose ships a ready Oracle Free container
export OCI_VISION_ENABLE_ORACLE=1 oci-vision analyze invoice.png --demo oci-vision search-runs "INV-1001"
05

Tech Stack

Python end to end. Typed, tested, and linted.

Stack Breakdown

Core

  • Pydantic v2 for response models
  • Typer + Rich for the CLI
  • Textual for the TUI cockpit
  • FastAPI + Jinja2 for the web dashboard

Extras

  • OCI SDK for live Vision calls
  • PIL / OpenCV for overlay rendering
  • python-oracledb for Oracle integration
  • pytest (200+ tests) for the test suite

Install Options

# One-command install curl -fsSL https://raw.githubusercontent.com/.../install.sh | bash # Or pick your extras pip install -e "." # demo CLI, cockpit, web pip install -e ".[live]" # + OCI SDK pip install -e ".[notebooks]" # + notebook tooling pip install -e ".[oracle]" # + Oracle DB support pip install -e ".[all]" # everything

The base install has zero heavy dependencies. Each extra pulls only what it needs.

CI & Security

  • build-install-smoke: package build, install flow, demo CLI paths, compare/batch, cockpit screenshots, web assets
  • dependency-audit: audits the base install path, uploads markdown report artifact on every run
